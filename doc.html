Introduction

Javascript is a cross-platform,object-orinented scripting language.its a small and lightweight language.Inside a host environment(for example, a web browser) javascript can be connected to the objects of its environment to provide programmatic control over them

javascipt contains a standard libary of objects,such as array,date and math,a core set of language elements such as operators,control,structures and statement.core javascript can be extended for a variety of purposes by supplementing it with additional objects;for example:

  • client-side javascript extends the core language by supplying objects to control a browser and its document object model(DOM).For example,client-side extensions allow an application to place elements on an HTML form and respond to user event such as mouse clicks,form input and page navigation.
  • server-side Javascript extwends the core language by supplying objects relevant to running Javascript on a server.for example,server-side extensions allow an application to communicate with a database,provide continuity of information from one invocation to another of the application,or perform file manipulation on a server.

    • javascript and Java

    Javascript and java are similar in some wayas but fundamentally different in some others. the javascript language resembles java but does not have java's static typing and strong type checking.javascript follows most of java expresssion syntax,naming convetions and basic control-flow constructs which was the reason why it was renamed from livescript to Javascript

    in contrast to java's compile-time system of classes built by declarations,javascript supports a runtime system based on a small number of data types representing numeric,boolean and string values.Javascript has a prototype-based object model instead of the more common class-based model provides dynamic inheritance,that is what is inherited can vary for individual objects.javascript also supports function without any special declarative requirements.Functions can be properties of objects,excuting as loosely typed methods.

    Javascript is a very free-form language compared to java.you do not have to declare all variables,classes and methods.you do not have to be concerned with whether methods are public,pirvate or protected and you dont have to implement interfaces.variables,parameters and functions return types are not explicity typed.

    Variables

    you use variables as symbolic names for values in your application.the names of variables,called identifers conform to certain rules a javascript identifer must start with a letter,underscore(_),or dollar sign($);subsquent characters can also be digits(0-9).because Javascript is case sensitive,letters include the characters"A"through "Z"(uppercase)and the characters"a" through"z"(lowercase). you can use ISO 8859-1 or unicode letters such as a and u in identifers.you can also use the unicode escape sequences as characters in identifers.some examp.les of legal names are number_hits,temp99 and _name.

    Declaring variables

    you can declare a variable in three ways: with the keyworld var.for example, var x =42 this syntax can be used to declare both local and global variables.by simply assigning it a value.for example, x = 42 this always declares a global variables.it generate a strict javascript warning.you shouldn't use this variant.with the keyworld let.for example let y =13 this syntax can be used to declare a block scope local variable.see varable scope below.

    Variable Scope

    when you declare a variable outside of any function,it is called a global variable because it is availabe to any other code in the current document.when you declare a variable within a function its a called a local variable because it is availabe only within that function

    javascript before ECMAscript 2015 does not have block statement scope;rather, a variable declared within a block is local to the function(or global scope)that the block resides within.for example the following code will log 5,because the scope of x is the function(or global context)within which x is declared,not the block,which in this case is an if statement

    if (true){ varx = 5 } console.log(x);//5 the behaviour changes,when using the let declaration introduced in ECMAscript2015. if(true){ let y = 5; } console.log(y); //ReferenceError:y is not defined
    Global variables

    gloabl variables are in fact properties ofthe global object.in the web pages the global object is window,so you can access global variables using the window variable syntax consequentlu, you can access gloabl variables declared in one window or frame from another window or frame by specifying the window or frame name.for example,if a variable called phoneNumber is declared in a document,you van refer to this variable from an iframe as parent.phoneNumber.
